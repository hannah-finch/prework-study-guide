HTML

  • The head element contains information about the webpage.
  • Meta data contains infomation for the browser and is not visible to the user.
  • The body element represents the visible content shown to the user.

CSS

  • * {...} to apply styles to everything
  • Apply styles to mulitple classes or id's with a comma between them

Git

  • git status: checks what branch we are currently on
  • git checkout main: switches your branch to main
  • git pull origin main: pulls the main branch so you can work with up-to-date code
  • git checkout -b branch-name(example: feature/add-html): creates a new branch and switches to it. (Now you can work)
  • git add -A: adds all the changed files in the working directory to the staging area of Git
  • git commit -m "description-of-changes": commits the files to your repo, with a message (-m) of what you changed
  • Check that your current branch is up to date before opening a pull request or pushing code (git pull origin main)
  • git push origin branch-name: pushes your changes to the repo
  • Merge your feature branch into the base branch on GitHub (main <-- feature) with a title and short description of changes
  • Once merged, the feature branch should be deleted, then update your local repo (git checkout main, then git pull origin main)

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • To execute a function, it must be called in the code.
